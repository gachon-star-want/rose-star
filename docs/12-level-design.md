# Level Design

## 목표

최종 릴리즈는 44개 해넘이(스테이지)다(D059).

```text
4 달 x 11 해넘이 = 44 stages
44 stages x Easy/Hard = 88 practical challenges
```

달 = 막(Act) = 바오밥 주력 성장단계. 각 해넘이는 30~90초 안에 끝나는 마이크로 방어 퍼즐이다.

## 큰 원칙

- 각 해넘이는 하나의 아이디어를 가르친다.
- 같은 맵을 Easy/Hard가 공유한다.
- Hard Mode에서도 클리어 가능해야 한다.
- Easy Mode는 초당 별빛으로 실수 여지를 늘린다.
- 긴 웨이브보다 짧고 밀도 있는 변주를 우선한다.
- 새 타워/적/규칙은 한 번에 하나씩 소개한다.
- 모든 단계의 바오밥은 상시 등장하되, 그 달의 주력 단계만 변동한다(D052).

## 해넘이 데이터 구조

해넘이는 다음 요소로 구성한다.

| 요소 | 설명 |
| --- | --- |
| Header | ID, 달, 해넘이, 사용 가능 타워, 시작 별빛, 장미 꽃잎, Easy 수입 |
| Path commands | 적 경로(장미까지) |
| Build zones | 설치 가능 영역 |
| Waves | 적 타입, 수, 간격, 지연 |
| Flags | Hard 팔레트, 특수 규칙 |

맵 전체 타일을 저장하지 않는다. 경로와 설치 영역을 명령으로 만든다.

## Header 초안

```text
id              db  ; 0..43
moon            db  ; 1..4 (달)
sunset          db  ; 1..11 (해넘이)
tower_mask      db  ; 사용 가능 타워 bitmask
start_light     dw  ; 시작 별빛
petals          db  ; 장미 꽃잎(생명)
easy_income     db  ; Easy Mode 초당 별빛
kill_reward_mul db  ; 처치 보상 배율
path_count      db
build_count     db
wave_count      db
flags           db
```

## 명령 초안

```text
SPAWN lane                       ; 이후 PATH들을 이 진입 lane 폴리라인에 귀속(합류 전까지). 생략 시 lane 0
PATH  x y len dir
BUILD x y w h
BLOCK x y w h
WAVE  type count spacing delay [lane]   ; lane = 이 무리가 나오는 진입(기본 0)
RULE  id value
```

대부분 템플릿은 진입 1개(lane 0)라 `SPAWN`/`lane`을 생략한다. **복수 진입 합류 맵(T6)만** `SPAWN 0`/`SPAWN 1`로 두 폴리라인을 선언하고, 각 폴리라인은 합류 셀에서 같은 꼬리(공유 tail)로 이어진다. WAVE의 `lane`이 그 무리의 진입을 고른다(엔진 요구사항은 §T6 참조).

방향 값:

| 값 | 방향 |
| --- | --- |
| 0 | Right |
| 1 | Down |
| 2 | Left |
| 3 | Up |

## 경로 템플릿 라이브러리 (확정 기하 — AI 구현 스펙)

해넘이마다 경로가 다르지만(1-1 직선·1-3 코너·2-4 S자…), 44개를 각각 손으로 그리는 대신 **재사용 템플릿 6종**을 좌표까지 확정하고, §44 해넘이 표가 해넘이별로 템플릿 ID를 배정한다(설계 결정 — D079). 같은 데이터를 여러 해넘이가 참조 → 정합·바이트 절약(exact-fill엔 무해). Easy/Hard는 같은 템플릿 공유.

**좌표계**: 그리드 16열(0~15) × 12행. **행 0 = HUD**(HUD_ROWS=1), 플레이 영역 = 행 1~11(16×11). 적은 좌측(열 0)으로 진입, **🌹장미는 각 경로 마지막 끝점(열 15)**. PATH 세그먼트는 끝점 셀을 다음 세그먼트와 공유한다. dir: 0R/1D/2L/3U.

### T0 STRAIGHT — 직선 (기준, 1-1 검증분)

```text
PATH  0 6 16 0          ; (0,6)→(15,6) 가로 직선, 장미 (15,6)
BUILD 2 4 12 1          ; 경로 위쪽 행4 (열2~13)
BUILD 2 8 12 1          ; 경로 아래쪽 행8 (열2~13)
```

### T1 LOWLINE — 낮은 직선 (위쪽 설치 공간 큼)

```text
PATH  0 9 16 0          ; (0,9)→(15,9), 장미 (15,9)
BUILD 2 7 12 1
BUILD 2 5 12 1          ; 위로 두 줄 → 사거리·중첩 학습용
```

### T2 LCORNER — 한 번 꺾임

```text
PATH  0 3 11 0          ; (0,3)→(10,3)
PATH 10 3  7 1          ; (10,3)→(10,9)
PATH 10 9  6 0          ; (10,9)→(15,9), 장미 (15,9)
BUILD 2 5 7 1           ; 가로구간 아래
BUILD 6 5 4 3           ; 꺾이는 모서리 안쪽(고가치 자리)
BUILD 11 7 4 1          ; 마지막 직선 위
```

### T3 SCURVE — S자 (두 코너)

```text
PATH  0 3  8 0          ; (0,3)→(7,3)
PATH  7 3  6 1          ; (7,3)→(7,8)
PATH  7 8  9 0          ; (7,8)→(15,8), 장미 (15,8)
BUILD 1 5 5 4           ; 좌하 빈 구역
BUILD 9 5 6 2           ; 우상 빈 구역(꺾임 안쪽)
```

### T4 DOUBLES — 이중 S (네 코너, 긴 경로)

```text
PATH  0 2  6 0          ; (0,2)→(5,2)
PATH  5 2  5 1          ; (5,2)→(5,6)
PATH  5 6  6 0          ; (5,6)→(10,6)
PATH 10 6  5 1          ; (10,6)→(10,10)
PATH 10 10 6 0          ; (10,10)→(15,10), 장미 (15,10)
BUILD 1 4 3 6           ; 좌측 세로 통로 옆
BUILD 6 8 3 2           ; 중앙 꺾임 사이
BUILD 11 7 4 2          ; 우측 꺾임 안쪽
```

### T5 SWITCHBACK — 스위치백 (가장 김, 네 코너 왕복)

```text
PATH  0 2 15 0          ; (0,2)→(14,2)
PATH 14 2  4 1          ; (14,2)→(14,5)
PATH 14 5 14 2          ; (14,5)→(1,5)   왼쪽으로 되돌아감
PATH  1 5  4 1          ; (1,5)→(1,8)
PATH  1 8 15 0          ; (1,8)→(15,8), 장미 (15,8)
BUILD 3 3 9 2           ; 위 왕복로 사이 띠(행3~4)
BUILD 3 6 9 2           ; 아래 왕복로 사이 띠(행6~7)
```

### T6 MERGE — 두 진입 합류 (맵 깊이 메커닉)

44 해넘이의 경로 다양성을 한 단계 끌어올리는 **복수 진입 합류** 템플릿. 위·아래 두 곳에서 적이 동시에 흘러나와 중앙 합류 셀에서 한 줄기로 모여 장미로 향한다 → "두 흐름을 한 병목으로 모아 막는" 새 퍼즐. 두 lane은 **합류까지 길이가 같고**(10칸) 그 뒤 **공유 tail**(8칸)을 함께 쓴다.

```text
SPAWN 0                 ; 위 진입(lane 0)
PATH  0 3 8 0           ; (0,3)→(7,3)
PATH  7 3 4 1           ; (7,3)→(7,6)  합류 셀
SPAWN 1                 ; 아래 진입(lane 1)
PATH  0 9 8 0           ; (0,9)→(7,9)
PATH  7 9 4 3           ; (7,9)→(7,6)  합류 셀(dir Up)
; 공유 tail (두 lane이 합류 후 함께 사용)
PATH  7 6 9 0           ; (7,6)→(15,6), 장미 (15,6)
BUILD 1 5 5 3          ; 좌중앙: 두 lane 사이 띠(행5~7) — 양쪽 동시 견제 고가치
BUILD 9 4 6 1          ; 합류 후 직선 위
BUILD 9 8 6 1          ; 합류 후 직선 아래
```

**엔진 요구사항(이 템플릿 전용, 다른 5종엔 영향 없음):**
- 템플릿당 폴리라인 ≤ `MAX_LANES=2`. 각 적은 `lane`(0/1) 필드를 갖고 자기 lane 폴리라인을 따라 이동. 공유 tail은 두 lane 폴리라인 데이터에 **각각 동일하게** 펼쳐 넣어(=합류 후 좌표가 일치) 분기 처리 없이 단순 추종 유지.
- **타게팅은 "장미까지 남은 거리 최소"로 측정**(`remaining = lane_len - traveled`). 두 lane이 합류 전 길이가 같고 tail을 공유하므로 lane이 달라도 비교가 일관 — path_progress(=traveled) 직접 비교도 성립하지만, lane 비대칭 변형을 대비해 *남은 거리* 기준으로 못박는다. `fire_tick`/SPLASH의 "가장 앞선 적"은 이 metric을 쓴다(10 §타게팅 보강).
- `PATH_END_FX`(leak 임계, docs/18)는 두 lane 공통(둘 다 열15에서 끝). spawn은 WAVE.lane이 가리키는 진입점 셀에서.
- 하네스: 두 lane 동시 진행도 결정론 유지(slot 오름차순 처리 그대로, lane은 위치 계산에만 영향).

이 한 메커닉만 추가하면 "동시 다발 → 병목 집중" 변주가 생겨 후반 해넘이의 전술 폭이 넓어진다(맵 깊이). 업그레이드 같은 밸런스 폭발 없이 *배치 퍼즐*만 깊어져 docs/08 리스크에 안전. 데이터 비용: lane 필드 1B/적 + 폴리라인 1벌 추가 ≈ 수십 B(exact-fill 무해).

### 빌드존 변형 (제한 설치 해넘이용)

"제한된 설치 칸"을 가르치는 해넘이(예: 1-10, 3-9)는 위 BUILD에서 일부만 연다. §44 표의 `build_mod` 열로 지정:

| build_mod | 의미 |
| --- | --- |
| `FULL` | 템플릿 기본 BUILD 전체 |
| `HALF` | 템플릿 BUILD 직사각형 중 **첫 1개만** 사용 |
| `SPOTS:(x,y);(x,y)…` | 명시한 개별 칸만 설치 가능(나머지 BLOCK) |

장미·경로 셀에는 어떤 모드여도 설치 불가(공통 규칙).

## 밸런스 기준값 (확정 초안)

AI 구현·밸런싱의 출발점이다(D063/D064). 해넘이별 웨이브 구성은 이 기준값 위에서 채운다.

### 다음 무리 앞당기기 — Hard의 능동 자원

**[N] `call_wave_early`**(docs/18 §spawn_tick)는 **Hard에서 유일한 능동 자원 창구**다. Easy는 초당 별빛 +2가 자원을 채워주지만, Hard는 수입이 처치 보상뿐 — 기다리는 동안 자원을 버는 방법이 없다. [N]으로 대기 웨이브를 앞당기면 남은 대기 시간에 비례한 보너스 별빛(`wave_timer / WAVE_BONUS_DIV = wave_timer / 30`)을 즉시 얻는다. 단, 다음 무리가 바로 쏟아지므로 타워 배치가 준비된 상태에서만 이득이 된다.

레벨 디자인 함의: **Hard 클리어 경로를 설계할 때 [N] 사용을 배치 계획에 포함**해야 한다. 특히 별빛이 빡빡한 중후반 해넘이(3-9 별빛 부족 등)는 [N] 타이밍을 의도해 짜는 것을 권장. 하네스 lock 시 [N] 호출 틱을 시나리오에 명시.

### 페이싱 가드레일

- **난이도 성향: 정통 TD답게 도전적(D069).** Hard는 진짜 어렵게, Easy도 무뇌 클리어는 아니다. 단 모든 해넘이는 **Hard에서도 이론상 클리어 가능**해야 한다(D064 시작 별빛+처치보상만으로) — 빡세되 불공정하지 않게.
- **"Hard 클리어 가능"은 책상에서 단언하는 속성이 아니라 lock 때 *증명*하는 목표다.** draft 숫자가 그 자체로 클리어를 보장하지 않는다 — 각 해넘이를 lock하려면 "초당 별빛 0(Hard)에서 시작 별빛+처치보상만으로 살 수 있는 타워 배치 *하나 이상*이 존재해 끝까지 막는다"를 하네스로 **구체적 배치와 함께** 보여야 한다(§lock 절차). 그 배치를 못 찾으면 그 해넘이는 미완 → 숫자를 낮추거나 설치 칸을 연다. 즉 클리어 가능성은 메서드(검증)이지 가정이 아니다.
- 어려움은 **적 수/체력/웨이브 밀도/병목(D047)·타게팅 압박**에서 온다. 적 체력 단순 뻥튀기로 만들지 않는다.
- 한 해넘이 30~90초. 웨이브는 적 타입 1~2종 중심(11 웨이브 조합 원칙).
- 달1은 🐑양 1종만으로 풀리도록(2-1에서 🌋화산 해금) — 11해넘이가 한 종으로 길지 않게 경로/병목(D047) 변주로 지루함 방지.
- 시작 별빛은 그 달 신규 타워를 1~2개 살 수 있는 선에서 ±10 조정. 도전 성향이라 여유를 과하게 주지 않는다.

## 44 해넘이 웨이브 스펙 (생성 제약 + draft 테이블)

웨이브 수치는 **밸런스 결과**라 책상에서 최종 확정 불가(D079) — 하네스 시뮬로 "Hard 클리어 가능"을 확인해야 진짜 값이 된다. 그래서 ① 지금 권위 있게 박는 **생성 제약식**과 ② 그 위에서 뽑은 **draft 숫자 44행**을, 각 행 **상태 태그**와 함께 싣는다. 콘텐츠 램프에서 한 행씩 하네스로 확인하고 `LOCKED`로 승격한다.

## 특수 모드 생성 알고리즘 (Endless·Random — 확정 알고리즘)

캠페인 44해넘이 밖의 두 보너스 모드(D057/D059)다. 둘 다 **절차 생성이라 데이터가 거의 0**(공식·생성기 코드만) → exact-fill에 무해, 오히려 콘텐츠 밀도↑. 생성은 모두 **결정론 PRNG 별도 인스턴스**(docs/18 §PRNG)로, 캠페인 ERRATIC 스트림과 섞지 않는다.

### 🏮Endless — 점등인의 별 (해금: 2-11 클리어)

고정 경로 + 무한 웨이브, **생존 웨이브 수 = 점수**(D057). 핵심 진실: **이 모드는 무한 승리가 불가능하다.** 설치 칸 유한 + 업그레이드 없음(D 본문) → 총 DPS에 상한이 있는데, 웨이브 위협은 끝없이 증가하므로 **언젠가 한 적을 경로 안에서 못 죽여 leak이 누적되고 꽃잎 5장이 소진된다.** "운이 따라주면 더 멀리" 가는 점수전이지, 영원히 버티는 모드가 아니다(Q2 답·D057 정합).

```text
맵: 전용 고정 템플릿 T_ENDLESS (설치칸 넉넉한 긴 경로 — 권장 T5 SWITCHBACK 재사용)
시드: 모드 진입 시각 기반 1회(별도 PRNG 인스턴스)
경제: 시작 별빛 80, 초당 별빛 0(Hard 경제 고정 — 점수전이라 난이도 단일), 꽃잎 5
처치 보상: §생성 제약식 reward 그대로

웨이브 w = 1,2,3,... (무한):
    budget(w)  = 10 + 3*w                      ; threat 예산 끝없이 증가
    hp_mul(w)  = 256 + 12*w   (÷256 적용)       ; 적 HP 배율(약 21웨이브마다 +100%)
    spacing(w) = max(24, 48 - w/4)             ; 점점 빽빽(하한 24틱)
    적 풀(해금 곡선 모사):
        w 1..10   : 싹·새싹
        w 11..20  : +묘목·뱀
        w 21..    : +거목·특수4종
    구성: budget을 적 풀에서 PRNG로 분배(threat 가중치로 마릿수 환산),
          동시 화면 ≤ MAX_ENEMIES(16)는 spacing으로 보장
점수 = 클리어한 웨이브 수(마지막으로 비운 w). 기록 코드는 모드 E/R 점수형(docs/18 §6).
```

> DPS 상한 증명(설계 근거): 설치칸 N개 × 최강 DPS 타워(가로등 5dmg/70f) = 고정 상한 D_max. 한 웨이브 처치요구 DPS는 hp_mul·count로 단조 증가 → ∃W: 요구 > D_max → 그 뒤 leak 불가피. 그래서 곡선은 "흑자(보상>비용)로 타워를 늘리는 전반 → hp_mul이 DPS를 추월하는 후반 → 불가피한 죽음"의 3국면. **무한 클리어가 아니라 도달 웨이브 경신이 목표**임을 UI에 명시(09).

### 🏜️Random — 사막의 신기루 (해금: 3-11 클리어)

매번 다른 **절차적 생성 맵(경로+빌드존) + 웨이브**(D057). 절차적 경로 생성은 *이 모드 한 곳에만 격리*된다(D028). 한 판은 캠페인 한 해넘이 분량의 유한 도전(생성된 웨이브 전멸 = 클리어). 시드가 매 진입 달라 레이아웃을 모른 채 적응하는 재미.

```text
시드: 모드 진입 시각 기반 1회(별도 PRNG). 같은 시드 = 같은 맵(리플레이/공유 가능)
경제: month3 수준(시작 별빛 130, Hard 경제), 적 풀 = 전 종 해금, budget = budget_base(3, rand 1..11)

gen_random_path(prng):                 ; drunkard walk, 좌측 진입 → 🌹우측 끝(열15)
    y = 2 + prng_next()%9              ; 진입 행 2..10
    path = []; x = 0; cur = (0,y)
    while x < 15:
        roll = prng_next()%10
        dir  = RIGHT if roll<6 else (UP if roll<8 else DOWN)   ; 우세=전진, 가끔 상하
        len  = 1 + prng_next()%3                                ; 1..3칸
        nxt  = step(cur, dir, len) clamp(행 1..11, 열 0..15)
        if 재방문(nxt 경유 셀) or 인접평행(경로 두께2 생김): 재롤(최대 8회, 실패 시 dir=RIGHT len=1)
        emit PATH 세그먼트(cur→nxt); cur = nxt; x = cur.x
    emit PATH (cur → (15, cur.y))      ; 마지막은 강제로 우측 끝 = 장미

gen_random_buildzones(path):
    free = 모든 비경로 셀
    BUILD = free 중 경로에 체비셰프≤2 인 셀만(사거리 닿는 자리) 직사각형 그리디 병합

솔버빌리티 휴리스틱(전부 통과해야 채택, 실패 시 seed+=1 후 재생성 — 결정론 재시도):
    H1 경로 길이 ≥ 16칸           (너무 짧으면 막을 틈 없음)
    H2 설치 가능 칸 ≥ 20          (배치 자유도)
    H3 경로 모든 셀이 어떤 BUILD 칸의 사거리(체비셰프≤3) 안   (사각지대 0 → 이론상 방어 가능)
    H4 연결·단일성: drunkard walk라 본래 보장(끊김·분기 없음)
```

> 연결성은 한 붓 그리기라 자동 보장. 진짜 관문은 H3(**사각지대 금지**) — 이게 "생성됐지만 못 깨는 맵"을 거른다. H1~H3 실패 시 seed++로 결정론 재생성하므로 무한 루프 없이 항상 풀 수 있는 맵만 낸다. Endless와 달리 유한이라 클리어/실패가 명확.

## 실패 기준

- Hard Mode로 이론상 클리어 불가능
- Easy Mode가 너무 쉬워 타워 배치 의미가 없음
- 새 요소가 설명 없이 여러 개 동시에 등장
- 한 해넘이가 90초를 자주 넘김
- 특정 타워 하나만 계속 정답이 됨
