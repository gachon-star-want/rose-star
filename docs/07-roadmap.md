# Roadmap

## 원칙

먼저 "작동하는 작고 재미있는 핵심"을 만든다. 그 다음 콘텐츠와 연출을 붙인다. 어셈블리어 프로젝트에서는 초반에 너무 많은 기능을 약속하면 완성 가능성이 급격히 낮아진다.

## Phase 0: 결정 고정

목표:

- 대상 플랫폼 확정
- 어셈블러/링커 확정
- 내부 해상도 확정
- 빌드 명령 확정

완료 조건:

- Windows PE `.exe` 형식 결정 완료
- Windows x86 PE 아키텍처 결정 완료
- FASM 어셈블러 결정 완료
- 256x192(4:3) 내부 해상도 결정 완료(D046)
- `make build` 또는 동등한 단일 빌드 명령 정의

## Phase 1: 부팅 가능한 화면

목표:

- FASM 확보
- 창 생성
- 프레임버퍼 표시
- 색 채우기
- 종료 입력 처리

완료 조건:

- 실행 파일이 생성됨
- 창이 뜸
- 256x192 프레임버퍼가 표시됨
- 60초 동안 크래시 없음
- `Esc`로 종료 가능
- 파일 크기 기록

## Phase 2: 커서와 입력

목표:

- 커서 표시
- 방향키 이동
- 설치 가능 칸 하이라이트
- 타워 설치/취소 입력

완료 조건:

- 조작 가능한 커서가 있음
- 설치 가능 칸과 불가능 칸이 구분됨
- 타워 1개를 설치할 수 있음

## Phase 3: 경로와 웨이브

목표:

- 타일맵 렌더링
- 스테이지 데이터 분리
- 고정 경로 처리
- 적 스폰/이동
- 도착점 피해 처리

완료 조건:

- 적이 경로를 따라 이동함
- 도착점에 닿으면 생명/바이트가 감소함
- 실패/재시작 가능

## Phase 4: 게임 루프

목표:

- 타이틀 또는 즉시 시작
- 타워 공격/명중
- 웨이브 클리어
- 다음 스테이지 이동
- 실패 후 재시작
- 최소 UI 표시

완료 조건:

- 3개 스테이지 연속 플레이 가능
- 게임을 처음 켠 사람이 규칙을 이해할 수 있음

## Phase 5: 44해넘이 콘텐츠

목표:

- 44개 마이크로 해넘이 제작(4달×11)
- 1차 타워 3종(양·화산·가로등), 최종 타워 5종 구현
- 1차 적 4종(바오밥 4단계), 최종 적 8종 구현
- 별빛/판매/웨이브 경제 구현
- 난이도 곡선 조정

완료 조건:

- 처음부터 끝까지 클리어 가능
- 4달 x 11해넘이 구조 완성
- 각 달이 다른 바오밥 성장단계/학습 테마를 가짐
- 각 해넘이가 하나의 명확한 아이디어를 가짐
- 달 단위 순차 해금 곡선이 적용됨
- 플레이 시간이 최소 30분 이상

## Phase 6: 오디오와 폴리시

목표:

- 설치/공격/피해/클리어 효과음
- 음악은 2차 목표로 유지
- 별·장미·바오밥 UI 연출(순수 우화, D049)
- 최종 이름과 화면 문구 정리

완료 조건:

- 사운드가 게임 피드백에 기여함
- 음악이 없어도 게임이 완성된 상태임
- 용량이 내부 상한을 넘지 않음

## Phase 7: 릴리즈 하드닝

목표:

- 최종 용량 측정
- 정확히 1,474,560바이트를 채우는 exact-fill 빌드 생성
- 실행 검증
- 불필요 파일 제거
- 제출 패키지 작성

완료 조건:

- 개발 안전 빌드는 `dist/` 기준 1,400,000바이트 이하
- 최종 릴리즈 빌드는 `dist/` 기준 정확히 1,474,560바이트
- 남는 바이트가 숨겨진 컨텐츠/메타데이터/리플레이 등 의미 있는 데이터로 채워짐
- 실제 대상 OS에서 실행 확인
- 첫 해넘이(1-1)와 마지막 해넘이 `4-11`(거목 보스) 클리어 확인
- 제출 폼 업로드 준비 완료
