# World Concept

## 한 문장 컨셉

**「어린왕자: 장미의 별」은 어린왕자의 작은 별 B-612에서, 장미 한 송이를 바오밥으로부터 지키는 초소형 픽셀 타워디펜스다.**

## 제품 메시지

- 1,474,560바이트를 정확히 채우는 게임
- 44개 해넘이(스테이지) = 4달 × 11
- Easy/Hard 모드로 실질 88개 챌린지
- 어셈블리어 기반 Windows x86 PE 단일 실행 파일
- 도트 그래픽, 팔레트 재사용, 명령 기반 레벨 데이터
- 원작 「어린왕자」 순수 우화 — 보이는 것만 레트로(D049)

## 세계관

플레이어는 어린왕자다. 자신의 작은 별 **B-612**에서 단 한 송이 **🌹장미**를 지킨다. 별 곳곳에서 **🌳바오밥**의 싹이 돋아 자라며 장미를 위협하고, 어린왕자는 별빛을 모아 수호자(타워)를 불러 길목을 막는다.

바오밥이 장미에 닿을 때마다 **꽃잎**이 한 장 떨어지고, 마지막 꽃잎이 지면 별을 잃는다. 바오밥은 작을 때(싹) 싸고 빠르게 처리할 수 있지만, 방치해 거목이 되면 막기 힘든 위협이 된다 — 원작의 교훈("바오밥은 작을 때 뽑아야 한다")을 그대로 메커닉으로 옮긴다.

특수한 위협들은 다른 별 어른들의 그림자다. 🐍뱀(빠른 암살자)·🎩허영쟁이(주변을 강하게)·💼사업가(별을 세어 소유, 보상 없음)·🍷술꾼(불규칙)이 B-612까지 번져온다.

세계관은 디지털 SF 액자 없이 순수 동화로 간다. 레트로(도트·팔레트·CRT 질감)는 시각 표현 레이어에만 적용한다(D049).

| 게임 용어 | 의미 |
| --- | --- |
| 달 (Moon) | 11해넘이 묶음. 4달 존재. 달=막=바오밥 주력 성장단계 |
| 해넘이 (Sunset) | 스테이지. 총 44개(4달×11). 44번째는 거목 보스 |
| 별빛 (Starlight) | 타워 소환·강화 자원 |
| 장미 꽃잎 (Rose Petal) | 생명/남은 안정성 |
| 바오밥 (Baobab) | 주적. 싹→새싹→묘목→거목 4단계 |
| 🌹장미 | 지켜야 할 코어 한 송이 |
| Easy Mode | 초당 별빛 생성 있음 |
| Hard Mode | 초당 별빛 생성 없음 |

## 캠페인 구조

| 달 | 주력 바오밥 | 해넘이 | 비고 |
| --- | --- | --- | --- |
| 달1 | 🌱싹 | 1~11 | |
| 달2 | 🌿새싹 | 12~22 | 22 클리어 시 Endless 해금 |
| 달3 | 🪴묘목 | 23~33 | 33 클리어 시 Random 해금 |
| 달4 | 🌳거목 | 34~44 | 44번째 = 거대 거목 보스 |

보너스 모드는 메인(B-612 정주 방어)과 별도 프레임을 쓴다(D057).

- **Endless = 🏮점등인의 별**: 자전이 빨라 쉴 새 없이 등을 켜고 끄는 끝없는 반복의 별. 고정 경로 + 무한 웨이브, 생존 웨이브 수가 점수.
- **Random = 🏜️사막의 신기루**: 매번 다른 모양으로 떠오르는 사막 길. 절차적 생성 맵(경로·빌드존 랜덤).

## 톤

작은 별 위의 조용한 우화. 지나치게 어둡거나 진지하지 않게, 그러나 장미를 지킨다는 책임의 무게는 남긴다.

텍스트는 짧고 기능적이어야 한다.

모든 카피는 **사람이 조용히 읊조리는 한국어**로 쓴다. UI식 명사 나열은 금지(D062).

좋은 문구:

- `별을 지켰다`
- `오늘도 별은 무사하다`
- `바오밥을 막았다`
- `별빛이 돌아왔다`

피할 문구:

- UI식 명사 나열(`해넘이 무사`, `달 1 끝`, `저장 성공` 같은 기계 로그 투)
- 긴 세계관 설명 / 튜토리얼 문장
- 원작 번역 문장 직접 인용(D048)
- 메뉴를 읽어야만 이해되는 규칙 설명

## UI 문자열 표 (확정)

AI 구현용 확정 문자열이다(D061/D062). 한글은 사람 말투(D062), 대괄호 단축키·코드·파일명만 ASCII. **이 표에 쓰인 음절이 곧 비트맵 폰트 글리프 서브셋이다.** `N`은 숫자 자리.

### 타이틀·메뉴

| 위치 | 문자열 |
| --- | --- |
| 타이틀 | `어린왕자: 장미의 별` |
| 시작 안내 | `아무 키나 누르면 시작한다` |
| 모드 선택 | `쉬움` / `어려움` |
| 쉬움 한 줄 | `별빛이 천천히 차오른다` |
| 어려움 한 줄 | `오직 네 손으로` |
| 진행 선택 | `달 1` … `달 4` / `해넘이 N-N` |

### HUD (상단 1행, 짧게)

| 항목 | 표기 | 비고 |
| --- | --- | --- |
| 자원 | `별빛 N` | |
| 생명 | `꽃잎 N` | |
| 위치 | `N-N` | 달-해넘이 |
| 웨이브 | `무리 N/N` | 현재/전체 |

### 전투 중 한마디 (토스트, 1~2초)

| 상황 | 문자열 |
| --- | --- |

| 바오밥을 막음 | `바오밥을 막았다` |
| 타워 판매 | `별빛이 돌아왔다` |

| 상황 | 문자열 |
| --- | --- |

| 위치 | 문자열 |

| 항목 | 표기 |

## 시각 방향

내부 해상도는 256x192(4:3)다(D046). 모든 시각 요소는 이 해상도에서 먼저 읽혀야 한다.

시각 원칙:

- 픽셀아트, 원작 삽화 기반 재해석(영화/애니판 모방 금지, D048)
- 제한 팔레트
- 8x8, 16x16 타일 중심
- 적 바디 3개 이하
- 타워 5종도 형태는 단순하게
- Easy/Hard는 새 이미지가 아니라 팔레트 변형으로 구분

## 모드 시각 차이

Easy Mode는 기본 팔레트를 사용한다.

Hard Mode는 게임 영역에 반전/고대비/위험 팔레트를 적용한다. 전체 화면을 무조건 반전하지 않는다. UI 텍스트와 기록 코드는 항상 읽을 수 있어야 한다.

권장 팔레트 방향:

| 모드 | 배경 | 경로 | 적 | UI |
| --- | --- | --- | --- | --- |
| Easy | 짙은 남청(밤하늘) | 모래빛/회색 | 짙은 갈색(바오밥) | 밝은 회색 |
| Hard | 거의 검정 | 보라/청록 반전 | 노랑/흰색 | 밝은 회색 |

## 화면 흐름

1. 타이틀 (「어린왕자: 장미의 별」)
2. 모드 선택: 쉬움/어려움
3. 달/해넘이 선택
4. 플레이
5. 결과
6. 공유 카드 저장 또는 다음 해넘이

1차 구현에서는 타이틀/선택 화면을 생략하고 바로 첫 해넘이를 시작해도 된다. 단, 최종 릴리즈에는 Easy/Hard 구분과 진행 구조가 보여야 한다.

## 공유 카드 컨셉

공유 카드는 플레이 결과를 담은 픽셀 포스터다.

핵심은 중앙의 큰 게임 스냅샷이다. 텍스트 기록은 주변에 배치한다.

필수 정보:

- `어린왕자: 장미의 별`
- `해넘이 N-N` (달-번호)
- `쉬움` 또는 `어려움`
- 남은 장미 꽃잎(`꽃잎 N`)
- 타워 수(`타워 N`)
- 판매 횟수(`판매 N`)
- 기록 코드(`코드 RS-...`)
