# World Concept

## 한 문장 컨셉

**「어린왕자: 장미의 별」은 어린왕자의 작은 별 B-612에서, 장미 한 송이를 바오밥으로부터 지키는 초소형 픽셀 타워디펜스다.**

## 제품 메시지

- 1,474,560바이트를 정확히 채우는 게임
- 44개 해넘이(스테이지) = 4달 × 11
- Easy/Hard 모드로 실질 88개 챌린지
- 어셈블리어 기반 Windows x86 PE 단일 실행 파일
- 도트 그래픽, 팔레트 재사용, 명령 기반 레벨 데이터
- 원작 「어린왕자」 순수 우화 — 보이는 것만 레트로(D049)

## 세계관

플레이어는 어린왕자다. 자신의 작은 별 **B-612**에서 단 한 송이 **🌹장미**를 지킨다. 별 곳곳에서 **🌳바오밥**의 싹이 돋아 자라며 장미를 위협하고, 어린왕자는 별빛을 모아 수호자(타워)를 불러 길목을 막는다.

바오밥이 장미에 닿을 때마다 **꽃잎**이 한 장 떨어지고, 마지막 꽃잎이 지면 별을 잃는다. 바오밥은 작을 때(싹) 싸고 빠르게 처리할 수 있지만, 방치해 거목이 되면 막기 힘든 위협이 된다 — 원작의 교훈("바오밥은 작을 때 뽑아야 한다")을 그대로 메커닉으로 옮긴다.

특수한 위협들은 다른 별 어른들의 그림자다. 🐍뱀(빠른 암살자)·🎩허영쟁이(주변을 강하게)·💼사업가(별을 세어 소유, 보상 없음)·🍷술꾼(불규칙)이 B-612까지 번져온다.

세계관은 디지털 SF 액자 없이 순수 동화로 간다. 레트로(도트·팔레트·CRT 질감)는 시각 표현 레이어에만 적용한다(D049).

| 게임 용어 | 의미 |
| --- | --- |
| 달 (Moon) | 11해넘이 묶음. 4달 존재. 달=막=바오밥 주력 성장단계 |
| 해넘이 (Sunset) | 스테이지. 총 44개(4달×11). 44번째는 거목 보스 |
| 별빛 (Starlight) | 타워 소환·강화 자원 |
| 장미 꽃잎 (Rose Petal) | 생명/남은 안정성 |
| 바오밥 (Baobab) | 주적. 싹→새싹→묘목→거목 4단계 |
| 🌹장미 | 지켜야 할 코어 한 송이 |
| Easy Mode | 초당 별빛 생성 있음 |
| Hard Mode | 초당 별빛 생성 없음 |

## 캠페인 구조

| 달 | 주력 바오밥 | 해넘이 | 비고 |
| --- | --- | --- | --- |
| 달1 | 🌱싹 | 1~11 | |
| 달2 | 🌿새싹 | 12~22 | 22 클리어 시 Endless 해금 |
| 달3 | 🪴묘목 | 23~33 | 33 클리어 시 Random 해금 |
| 달4 | 🌳거목 | 34~44 | 44번째 = 거대 거목 보스 |

보너스 모드는 메인(B-612 정주 방어)과 별도 프레임을 쓴다(D057).

- **Endless = 🏮점등인의 별**: 자전이 빨라 쉴 새 없이 등을 켜고 끄는 끝없는 반복의 별. 고정 경로 + 무한 웨이브, 생존 웨이브 수가 점수.
- **Random = 🏜️사막의 신기루**: 매번 다른 모양으로 떠오르는 사막 길. 절차적 생성 맵(경로·빌드존 랜덤).

## 서사 (게임 표면, 초안 — D068/D071)

서사는 **빙산형**이다(D071): 깊은 백그라운드 lore는 `docs/16-lore-bible.md`에 두고(소설 수준까지 길어도 됨), **게임 속에는 "개연성을 채울 정도"의 최소한만** 노출한다. 아래가 그 표면이다 — 깊은 lore에서 길어 올린 짧은 문장들.

원작 어린왕자를 충실히 따르되, 떠나는 대신 **남아서 장미를 지킨다**로 재해석한다(D056 정주 방어). 문체는 절제·서정 — 짧고 여운 있게(사람 말투, 원작 번역 인용 없이 자작 — D048). 도트 장면 + 짧은 글로 보여주고, 큰 일러스트 컷신은 쓰지 않는다.

### 도입 (첫 시작)

> 내겐 별이 하나 있다. 작은 별, 그리고 내가 길들인 장미 한 송이.
> 별엔 바오밥 씨가 돋는다. 작을 땐 아무것도 아니지만, 두면 뿌리가 별을 쪼갠다.
> 다른 별로 떠날 수도 있었다. 그러나 나는 남기로 했다 — 장미 곁에서, 바오밥을 뽑으며.

### 막간 1 (달1→달2, 싹→새싹)

> 한 달이 저물었다. 싹들은 새싹이 되었다.
> 멀리, 다른 별의 그림자가 어른거린다.

### 막간 2 (달2→달3, 새싹→묘목)

> 황금빛 뱀이 모래 위를 지난다.
> 헛됨도, 셈도, 취함도 — 어른들의 것이 내 별까지 왔다.
> 나는 장미를 본다. 아직 피어 있다.

### 막간 3 (달3→달4, 묘목→거목)

> 가장 오래된 바오밥이 하늘을 가린다.
> 여우가 일러준 걸 떠올린다 — 길들인 것은 끝까지 책임지는 거라고.
> 마지막 밤이다.

### 엔딩 — 별을 지켰다 (4-11 클리어)

> 거목이 쓰러졌다. 별은 조용하다.
> 장미가 웃는다. 나도 웃는다.
> 이제 밤하늘을 보면, 어느 별 하나에서 장미가 피어 있을 것이다.
> 그럼 모든 별이 웃는 것처럼 보이겠지.

### 엔딩 — 별이 어두워졌다 (패배)

> 마지막 꽃잎이 졌다. 별이 어두워졌다.
> … 다시 해볼까.

*(초안 — 사용자 승인/수정 후 확정. 막간은 해당 달 첫 해넘이 진입 시 표시.)*

## 톤

작은 별 위의 조용한 우화. 지나치게 어둡거나 진지하지 않게, 그러나 장미를 지킨다는 책임의 무게는 남긴다.

텍스트는 짧고 기능적이어야 한다.

모든 카피는 **사람이 조용히 읊조리는 한국어**로 쓴다. UI식 명사 나열은 금지(D062).

좋은 문구:

- `별을 지켰다`
- `오늘도 별은 무사하다`
- `바오밥을 막았다`
- `별빛이 돌아왔다`

피할 문구:

- UI식 명사 나열(`해넘이 무사`, `달 1 끝`, `저장 성공` 같은 기계 로그 투)
- 긴 세계관 설명 / 튜토리얼 문장
- 원작 번역 문장 직접 인용(D048)
- 메뉴를 읽어야만 이해되는 규칙 설명

## UI 문자열 표 (확정)

AI 구현용 확정 문자열이다(D061/D062). 한글은 사람 말투(D062), 대괄호 단축키·코드·파일명만 ASCII. **이 표에 쓰인 음절이 곧 비트맵 폰트 글리프 서브셋이다.** `N`은 숫자 자리.

### 타이틀·메뉴

| 위치 | 문자열 |

| 타이틀 | `어린왕자: 장미의 별` |

| 모드 선택 | `쉬움` / `어려움` |

## 시각 방향

내부 해상도는 256x192(4:3)다(D046). 모든 시각 요소는 이 해상도에서 먼저 읽혀야 한다.

시각 원칙:

- 픽셀아트, 원작 삽화 기반 재해석(영화/애니판 모방 금지, D048)
- 제한 팔레트
- 8x8, 16x16 타일 중심
- 적 바디 3개 이하
- 타워 5종도 형태는 단순하게
- Easy/Hard는 새 이미지가 아니라 팔레트 변형으로 구분

## 아트 레퍼런스 & 가독성 팔레트 (D070/D072)

방향은 "밤하늘 동화"지만 **가독성이 분위기보다 우선**이다(D072). 어두운 배경 + 밝고 채도 높은 전경으로 이 둘을 동시에 잡는다.

### 레퍼런스 (모방 아닌 참고 — D048)

- **Kingdom: Two Crowns** — 밤/어둠을 쓰면서도 **실루엣 + 따뜻한 빛 포인트**로 유닛이 또렷이 읽히는 정석. "어두운데 잘 보이는" 우리 목표의 대표 예. (https://www.kingdomthegame.com/kingdom-two-crowns)
- **itch.io high-contrast + pixel-art 태그** — 접근성(고대비) 기준으로 모인 도트 게임 풀. 가독성 레퍼런스로 좋음. (https://itch.io/games/accessibility-highcontrast/tag-pixel-art)
- **CraftPix 탑다운 TD 픽셀 세트** — 타워/적/타일의 TD용 읽힘 처리 참고(우리 16x16 단순 실루엣 기준). (https://craftpix.net/sets/tower-defense-top-down-pixel-art/)

### 가독성 팔레트 초안

| 역할 | 방향 | 의도 |
| --- | --- | --- |
| 배경(밤하늘) | 짙은 남청, 채도 낮게 | 후퇴 — 전경을 살림 |
| 경로 | 밝은 모래빛/회청, 명도 대비 확실 | 길이 한눈에 |
| 빌드존 | 배경보다 한 톤 밝게 + 외곽선 | 놓을 곳 즉독 |
| 별빛(자원) | 채도 높은 노랑 | HUD·획득 즉시 인지 |
| 타워(아군) | 밝고 채도 높게(양=크림/흰 등) | 전진 — 내 편 식별 |
| 바오밥(적) | **어두운 갈색이라도 밝은 림/외곽 강조** | 어두운 배경에 묻히지 않게 |
| 위험/피해 | 붉음·핫핑크 포인트 | 경고 즉독 |

핵심 규칙: **적·타워는 색을 빼도 실루엣만으로 구별**되어야 하고, 어떤 적도 밤배경에 묻히면 안 된다(외곽 강조 허용). HUD 텍스트·기록 코드는 항상 읽힘(09 톤). 바오밥이 짙은 갈색이라 남청 배경에 묻힐 위험이 가장 크니 림 라이트/명도 분리를 최우선으로 본다.

> 에셋은 Executor가 16×16 스프라이트를 팔레트 인덱스·비트패턴 데이터로 직접 작성한다(D073, 외부 AI 생성 안 씀). 가독성(D072)은 작성 단계에서 직접 보장 — 첫 작성부터 밤배경에 안 묻히게.

## 모드 시각 차이

Easy Mode는 기본 팔레트를 사용한다.

Hard Mode는 게임 영역에 반전/고대비/위험 팔레트를 적용한다. 전체 화면을 무조건 반전하지 않는다. UI 텍스트와 기록 코드는 항상 읽을 수 있어야 한다.

권장 팔레트 방향:

| 모드 | 배경 | 경로 | 적 | UI |
| --- | --- | --- | --- | --- |
| Easy | 짙은 남청(밤하늘) | 모래빛/회색 | 짙은 갈색(바오밥) | 밝은 회색 |
| Hard | 거의 검정 | 보라/청록 반전 | 노랑/흰색 | 밝은 회색 |

5. **막간 장면**(각 달 첫 해넘이 진입 시) — 서사 막간 1~3
6. 플레이
7. 결과
8. 공유 카드 저장 또는 다음 해넘이
9. **엔딩 장면**(4-11 클리어/패배 시) — 서사 엔딩

서사 장면은 도트 배경 + 짧은 글, 아무 키/클릭으로 넘긴다. 1차 구현에서는 타이틀/선택/서사 화면을 생략하고 바로 첫 해넘이를 시작해도 된다. 단, 최종 릴리즈에는 Easy/Hard 구분과 진행 구조가 보여야 한다.

## 공유 카드 컨셉

공유 카드는 플레이 결과를 담은 픽셀 포스터다.

핵심은 중앙의 큰 게임 스냅샷이다. 텍스트 기록은 주변에 배치한다.

필수 정보:

- `어린왕자: 장미의 별`
- `해넘이 N-N` (달-번호)
- `쉬움` 또는 `어려움`
- 남은 장미 꽃잎(`꽃잎 N`)
- 타워 수(`타워 N`)
- 판매 횟수(`판매 N`)
- 기록 코드(`코드 RS-...`)
