## 현재 장르

2D 고정 경로 기반 마이크로 타워디펜스.

초기 Mario식 문법을 게임 장르로 따라가지 않는다. 대신 작은 데이터, 강한 재사용, 명령 기반 레벨 구성, 많은 스테이지 밀도를 참조한다.

## 제목

**어린왕자: 장미의 별** (D058)

순수 어린왕자 우화 톤(D049). 어린왕자가 자신의 별 B-612에서 장미 한 송이를 바오밥으로부터 지킨다. 기존 가제 "Floppy Defense"는 폐기.

## 핵심 구조

최종 릴리즈 목표는 **44개 해넘이(스테이지)** 다(D059).

44개는 **4달 x 11해넘이** 구조로 묶는다. 달 = 막(Act) = 바오밥 주력 성장단계.

```text
달1 🌱싹  : 해넘이 1-1 ... 1-11
달2 🌿새싹: 해넘이 2-1 ... 2-11
달3 🪴묘목: 해넘이 3-1 ... 3-11
달4 🌳거목: 해넘이 4-1 ... 4-11 (4-11 = 거목 보스)
```

각 해넘이는 30~90초 안에 끝나는 짧은 방어 퍼즐이다. 하나가 길어지는 것보다, 작은 규칙 변주를 44개 쌓는 쪽이 이 프로젝트와 맞다.

Easy/Hard 모드를 나누므로 체감상 각 해넘이는 두 가지 챌린지를 가진다.

```text
44 stages x 2 modes = 88 practical challenges
```

단, 데이터는 88개 맵으로 복제하지 않는다. 같은 해넘이에 대해 Easy Mode는 초당 별빛을 지급하고, Hard Mode는 지급하지 않는다.

Hard Mode는 시각적으로도 즉시 구분되어야 한다. 가장 싼 방법은 팔레트 변형이다.

- Easy Mode: 기본 팔레트
- Hard Mode: 반전/고대비/위험 팔레트

완전한 화면 색상 반전은 구현은 쉽지만 UI/기록 코드 가독성을 해칠 수 있다. 따라서 권장 방식은 전체 픽셀을 무조건 반전하기보다, 게임 영역 팔레트만 Hard 변형으로 바꾸는 것이다. UI 텍스트는 읽기 쉬운 색을 유지한다.

## 화면 구성

내부 해상도: **256x192 확정 (4:3)**. (D046 — 기존 256x144/16:9를 대체)

표시 방식: **창 모드 전용 + 정수배 스케일(기본 4배 = 1024x768)**. 풀스크린은 지원하지 않는다. 정수배라 픽셀이 항상 선명하고 어셈블리 구현이 단순하다.

기본 그리드:

| 항목 | 값 |
| --- | --- |
| 내부 해상도 | 256x192 (4:3) |
| 타일 크기 | 16x16 |
| 화면 타일 수 | 16x12 |
| 플레이 영역 | 16x11 (상단 UI 1행 제외) |
| UI 높이 | 16px (상단 1행) |
| 표시 방식 | 창 모드 전용, 정수배 4배(1024x768) |
| 기본 타워 크기 | 16x16 |
| 기본 적 크기 | 8x8 또는 12x12 |

기본 화면 요소:

- 경로
- 타워 설치 가능 칸
- 적 웨이브
- 현재 달/해넘이 번호
- 남은 장미 꽃잎
- 보유 별빛

별도 튜토리얼 화면·설명 텍스트를 만들지 않는다(D065). 슈퍼마리오 1-1처럼 **플레이하면서 조작·규칙을 자연스럽게 익히도록** 달1(첫 11해넘이)을 설계한다 — 1-1은 🐑양 1종·짧은 경로·약한 🌱싹 소수로 "설치→방어"가 저절로 이해되고, 이후 새 요소를 한 번에 하나씩만 도입한다. 허용되는 안내는 짧은 키 힌트(09)뿐이다.

## 조작

키보드와 마우스를 **둘 다 기본 입력**으로 지원한다(D066). 어느 쪽만으로도 플레이 가능하다.

| 동작 | 마우스 | 키보드 |
| --- | --- | --- |
| 커서 이동 | 마우스 이동 | 방향키 |
| 타워 설치 / 선택 | 좌클릭 | Z / Space / Enter |
| 취소 / 타워 판매 | 우클릭 | X / Backspace |
| 타워 종류 전환 | 상단 UI 아이콘 클릭 | Tab |
| 현재 해넘이 재시작 | — | R |
| 종료 | — | Esc |

마우스 좌표는 16x16 타일에 매핑한다(Win32 `WM_MOUSEMOVE`/`WM_LBUTTONDOWN`/`WM_RBUTTONDOWN`). 키보드 동등 매핑을 항상 유지해 접근성과 테스트 단순성을 확보한다.

## 핵심 루프

1. 해넘이 시작
2. 플레이어가 제한된 별빛으로 타워를 배치
3. 웨이브 시작
4. 적이 고정 경로를 따라 이동
5. 타워가 사거리 안의 적을 자동 공격
6. 적이 장미에 도달하면 장미 꽃잎 1장 감소
7. 모든 웨이브를 막으면 클리어
8. 실패하면 즉시 재시작

## 타워

타워 5종은 D054에서 확정. 역할별 1:1 분리(단일/AoE/장거리/슬로우/버프).

1차 목표 타워는 3종으로 제한한다.

| 타워 | 역할 | 특징 |
| --- | --- | --- |
| 🐑양 | 기본 단일 근접 | 싸고 안정적, 바오밥 천적 |
| 🌋화산 | 범위 공격(AoE) | 무리 대응 |
| 💡가로등 | 장거리 저격 | 단일 강타, 방어력 높은 적 대응 |

최종 목표 타워는 5종이다.

| 타워 | 역할 | 특징 |
| --- | --- | --- |
| 🐑양 | 기본 단일 근접 | 싸고 안정적, 바오밥 천적 |
| 🌋화산 | 범위 공격(AoE) | 무리 대응 |
| 💡가로등 | 장거리 저격 | 단일 강타, 방어력 높은 적 대응 |
| 🏜️사막바람 | 슬로우 | 빠른 적과 긴 경로에서 강함 |
| 🦊여우 | 지원/버프 | 주변 타워 공격속도 또는 사거리 증가 |

타워 수를 무작정 늘리지 않는다. 최종 5종은 단일/범위/장거리/슬로우/지원 역할을 분리하기 위한 상한이다. (🐑양에 "장미도 먹을 수 있는" 리스크 메커닉 여지 — D054)

## 적

적은 **리소스를 늘리지 않고 종류를 늘리는 방식**으로 설계한다.

도트 그래픽이 목표이므로 새 적을 만들 때마다 새 큰 이미지를 만들지 않는다. 8x8 또는 16x16 바디 타일, 팔레트, 행동 플래그, 속도/체력/보상 수치 조합으로 적을 다양화한다.

스프라이트 리소스 목표:

| 리소스 | 목표 |
| --- | --- |
| 적 바디 | 3개 이하 |
| 적 팔레트 | 4~8개 |
| 애니메이션 | 2프레임 중심 |
| 특수 효과 | 점멸, 색 교체, 작은 파티클 |

적 8종은 D053에서 확정. 바오밥 4성장단계 + 특수 4종.

1차 구현 적은 바오밥 단계 중심 4종으로 제한한다.

| 적 | 역할 | 특징 |
| --- | --- | --- |
| 🌱싹 | 기본 적 | 빠르고 약함, 싸게 처리 |
| 🌿새싹 | 평균 | 평균 속도/체력 |
| 🪴묘목 | 준탱커 | 느려지고 체력 상승 |
| 🌳거목 | 탱커 | 매우 느림, 높은 체력, 큰 보상 |

최종 목표 적은 8종이다.

| 적 | 바디 | 변주 방식 | 역할 |
| --- | --- | --- | --- |
| 🌱싹 | Small | 기본 팔레트 + 빠름 | 기본/약함 |
| 🌿새싹 | Small | 평균 팔레트/수치 | 평균 |
| 🪴묘목 | Large | 어두운 팔레트 + 높은 체력 | 준탱커 |
| 🌳거목 | Large | 큰 바디 + 매우 느림 | 탱커/보스 |
| 🐍뱀 | Small | 빠른 속도 + 사거리 회피 | 암살자(장미 직행) |
| 🎩허영쟁이 | Small | 오라 플래그 | 주변 바오밥 강화 |
| 💼사업가 | Small | 보상 0 플래그 | 처치 보상 없음/자원 교란 |
| 🍷술꾼 | Small | 불규칙 이동 + 슬로우 저항 | 명중/예측사격 교란 |

확장 목표:

- 변종은 새 바디 리소스를 늘리기 전에 팔레트/수치/플래그 조합을 먼저 사용한다.

## 경로 모델

경로 모델은 **고정 경로**로 확정한다.

- 플레이어가 경로를 막거나 새 길을 만들지 않는다.
- 해넘이 데이터가 경로를 직접 가진다.
- 적은 경로 노드 배열을 따라 이동한다.
- 타워는 경로 밖의 설치 가능 칸에만 놓는다.

이 선택은 중요하다. 자유 배치/동적 길찾기는 타워디펜스답지만 어셈블리 구현 복잡도가 크게 올라간다. 이 프로젝트는 44해넘이 밀도와 1.44MB exact-fill 메시지가 더 중요하다.

동적 길찾기는 비목표다. 모든 해넘이는 경로를 명시적으로 가진다. 플레이어는 경로 위에 타워를 놓을 수 없고, 타워 배치로 적 경로를 막거나 바꾸지 않는다. (절차적 경로 생성은 Random 모드 한 곳에만 격리 — D028/D057)

## 해넘이 구조

최종 스테이지 수:

**44개 해넘이 확정 = 4달 × 11.** (D059)

개발 마일스톤:

| 단계 | 해넘이 수 | 목적 |
| --- | ---: | --- |
| Prototype | 1 | 경로/타워/적/승패 루프 검증 |
| Alpha | 11 | 달1 기본 타워/적/경제 튜토리얼 |
| Beta | 22 | 달1~2 난이도 곡선과 데이터 형식 검증 |
| Release | 44 | 최종 4달×11 구조 완성 |

각 달은 주력 바오밥 성장단계를 한 칸씩 올리며, 그 안에서 학습 테마를 푼다(D052/D059). 모든 단계는 상시 등장하되 주력만 변동.

| 달 | 주력 | 학습 테마(예) |
| --- | --- | --- |
| 달1 | 🌱싹 | 🐑양, 기본 경로, 빠르고 약한 싹 처리 |
| 달2 | 🌿새싹 | 🌋화산·💡가로등, 경로 변주, 무리/장거리 |
| 달3 | 🪴묘목 | 🏜️사막바람, 탱커화 대응, 속도 제어 |
| 달4 | 🌳거목 | 🦊여우, 특수 4종 종합, 거목 보스 |

각 해넘이는 하나의 아이디어만 명확히 가르친다. 44개를 채우기 위해 긴 웨이브를 늘리지 않는다.

## 해금 곡선

44해넘이는 처음부터 모든 타워와 적을 던지지 않는다. 각 달은 새로운 선택지를 하나씩 풀고, 이전 선택지를 다시 조합한다.

| 구간 | 해금/강조 | 목적 |
| --- | --- | --- |
| 1-1 | 🐑양, 🌱싹 | 기본 설치와 경로 이해 |
| 1-6 | 🐍뱀 | 빠른 암살자 소개 |
| 2-1 | 🌋화산, 🌿새싹 | 범위 공격과 무리 대응 |
| 2-6 | 💡가로등 | 장거리 저격과 위치 가치 |
| 3-1 | 🪴묘목 | 탱커화와 누적 피해 |
| 3-6 | 🏜️사막바람 | 속도 제어 |
| 4-1 | 🦊여우, 🎩허영쟁이 | 버프 타워와 강화 오라 대응 |
| 4-6 | 💼사업가·🍷술꾼 | 경제 교란·명중 교란 대응 |
| 4-11 | 전체 조합 | 거목 보스 종합 |

보너스 해금: **Endless = 2-11(22번째) 클리어**, **Random = 3-11(33번째) 클리어** (D057/D059).

해금은 하드 락일 수도 있고, 튜토리얼상 강제 제공일 수도 있다. 구현은 단순하게 하기 위해 해넘이별 사용 가능 타워 bitmask를 둔다.

## 경제

권장 경제 모델:

- 시작 별빛: 해넘이별 고정
- 적 처치 보상: 소량 별빛
- 타워 판매: 구매가의 60% 환급
- Hard Mode: 초당 별빛 생성 없음
- Easy Mode: 초당 n 별빛을 지급하는 완화 모드
- 웨이브 보너스: 1차 목표에서는 제외

자원 이름은 **별빛(Starlight)** 으로 확정(D055). 적을 처치하면 갇힌 별빛이 하늘로 돌아오고, 그 별빛으로 수호자를 부른다.

이 프로젝트에서 "이자"는 은행 이자처럼 보유 자원 비례 보상이 아니다. 난이도 조절용 **초당 별빛 생성**이다.

| 모드 | 별빛 흐름 | 의도 |
| --- | --- | --- |
| Hard Mode | 시작 별빛 + 처치 보상만 사용 | 빡센 기본/챌린지 풀이 |
| Easy Mode | 시작 별빛 + 처치 보상 + 초당 n 별빛 | 덜 빡센 완화 모드 |

중요한 원칙:

- 모든 해넘이는 Hard Mode에서도 클리어 가능해야 한다.
- Easy Mode는 클리어 불가능을 가능하게 만드는 치트가 아니라, 실수 여지를 늘리는 난이도 조절 장치다.
- 해넘이 데이터는 시작 별빛, 처치 보상 배율, 초당 지급량을 명시할 수 있어야 한다.
- 점수/클리어 기록에는 Easy/Hard를 구분해 표시한다.

## 점수와 메타 규칙

게임 점수와 용어는 우화 세계관과 연결한다(D060).

- 생명: 장미 꽃잎
- 자원: 별빛
- 스테이지: 해넘이
- 묶음/막: 달
- 적: 바오밥(외 특수 4종)
- 클리어: "별을 지켰다" (전체 UI 문자열은 09 "UI 문자열 표")

## 기록과 공유

해넘이 클리어 기록은 Easy Mode와 Hard Mode를 구분한다.

기록 항목 후보:

- 모드: Easy/Hard
- 해넘이: 달-번호
- 남은 장미 꽃잎
- 사용한 타워 수
- 판매 횟수
- 클리어 시간 또는 웨이브 종료 프레임
- 상태 체크섬

온라인 글로벌 리더보드는 1차 목표에서 제외한다. 서버, 네트워크, 오류 처리, 개인정보/닉네임 처리, 부정 기록 검증이 필요해 1.44MB 어셈블리 프로젝트의 핵심과 멀다.

대신 1차/2차 후보로 **공유 가능한 기록 코드**를 둔다.

예시:

```text
RS-4-11-H-07-12-03-A91F
```

의미:

- `RS`: Rose's Star (어린왕자: 장미의 별)
- `4-11`: 해넘이(달-번호)
- `H`: Hard Mode
- `07`: 남은 장미 꽃잎
- `12`: 사용한 타워 수
- `03`: 판매 횟수
- `A91F`: 체크섬

이 방식은 사용자가 스크린샷이나 텍스트로 기록을 공유할 수 있고, 네트워크 없이도 구현 가능하다.

## 공유 카드

Instagram 같은 외부 서비스에 직접 업로드하는 기능은 목표로 두지 않는다. 로그인, 네트워크, 브라우저/앱 연동, 실패 처리, 정책 변경 대응이 필요해 단일 exe 게임의 범위를 벗어난다.

대신 결과 화면에서 한 번의 입력으로 **공유 카드 이미지**를 저장한다.

권장 동작:

- 결과 화면에서 `S` 입력
- 즉시 현재 폴더 또는 `share/` 폴더에 공유 카드 저장
- 저장 성공/실패를 짧게 표시
- 기록 코드 클립보드 복사는 부가 기능
- 짧은 공유 캡션 `.txt` 저장은 부가 기능

권장 이미지:

| 항목 | 값 |
| --- | --- |
| 1차 목표 | 1080x1080 정사각형 공유 카드 |
| 2차 목표 | 1080x1920 세로 스토리 카드 |
| 스타일 | 픽셀 아트, 제한 팔레트, 가운데 큰 게임 스냅샷 |
| 내용 | 게임 스냅샷, 게임명, 해넘이, Easy/Hard, 남은 장미 꽃잎, 타워 수, 판매 횟수, 기록 코드 |

파일 형식:

- 1차/릴리즈 안전 목표: BMP
- 2차/보너스 목표: 초소형 PNG encoder
- 외부 PNG 라이브러리: 금지

BMP는 Windows에서 만들기 쉽고 구현 코드가 작다. 1080x1080 BMP 파일은 실행 후 생성되는 산출물이므로 제출물 1.44MB 제한에 포함되지 않는다. PNG는 Instagram 업로드에 더 무난하지만 인코더 구현량이 있으므로, 용량과 일정 여유가 있을 때만 추가한다.

핵심 UX 원칙:

**버튼 하나를 누르면 바로 이미지 파일이 생겨야 한다.**

공유 카드 저장은 메뉴 깊숙한 곳에 숨기지 않는다. 결과 화면에서 바로 실행 가능해야 한다.

## 안내 팝업

결과 화면에는 공유 기능을 발견할 수 있도록 작은 안내 팝업을 둔다.

원칙:

- 긴 설명문을 넣지 않는다.
- 키 힌트 중심으로 보여준다.
- 결과 화면 진입 직후 2~3초 동안 표시하거나, 하단에 작게 상시 표시한다.
- 한 번 공유를 실행하면 성공/실패 메시지를 1초 정도 표시한다.

예시:

```text
[S] 카드로 남기기   [C] 코드 복사   [R] 다시   [Esc] 나가기
```

저장 성공:

```text
카드를 남겼다   share/RS-4-11-H-A91F.bmp
```

저장 실패:

```text
카드를 남기지 못했다
```

예시 캡션:

```text
어린왕자: 장미의 별 / 해넘이 4-11 / 어려움 / 꽃잎 07 / 코드 RS-4-11-H-07-12-03-A91F
```

공유 카드 레이아웃:

```text
┌──────────────────────────────┐
│ 어린왕자: 장미의 별          │
│ 해넘이 4-11         어려움   │
│                              │
│   ┌──────────────────────┐   │
│   │                      │   │
│   │   GAME SNAPSHOT      │   │
│   │   256x192 scaled     │   │
│   │                      │   │
│   └──────────────────────┘   │
│                              │
│ 꽃잎 07          타워 12     │
│ 판매 03                      │
│ 코드 RS-4-11-H-07-12-03-A91F │
└──────────────────────────────┘
```

스냅샷 원칙:

- 기본은 클리어 순간의 결과 화면 직전 256x192 프레임버퍼를 사용한다.
- Hard Mode 스냅샷은 Hard 팔레트 변형이 보이도록 저장한다.
- 1080x1080 카드 안에서 정수 배율로 확대한다.
- 픽셀 가장자리가 흐려지지 않게 nearest-neighbor 확대를 사용한다.
- 카드 가운데에 가장 큰 시각 요소로 배치한다.
- 기록 정보는 스냅샷을 가리지 않는다.

## 오디오 피드백

1차 목표에는 효과음만 포함한다.

| 이벤트 | 목적 |
| --- | --- |
| 타워 설치 | 입력 확인 |
| 공격/명중 | 전투 피드백 |
| 적 도달 | 피해 인지 |
| 해넘이 클리어 | 짧은 성취감 |

음악은 2차 목표다. 음악이 없어도 게임은 완성되어야 한다. 단, 최종 exact-fill 단계에서는 남는 바이트를 음악 패턴이나 숨겨진 트랙으로 채우는 것을 우선 검토한다.

## 난이도 원칙

- 성향은 **정통 TD답게 도전적(D069)** — Hard는 진짜 어렵게, Easy도 무뇌 클리어는 아니다. 단 모든 해넘이는 Hard에서도 이론상 클리어 가능(D064, 빡세되 불공정 금지).
- 실패 원인이 경로/사거리/별빛 중 무엇인지 명확해야 한다.
- 달1(첫 11해넘이)은 튜토리얼이다.
- 한 해넘이는 30~90초 안에 끝난다.
- 어려움은 적 체력 뻥튀기가 아니라 배치 판단에서 와야 한다.
- 마지막 해넘이도 암기만 요구하지 말고 배치 최적화를 요구해야 한다.
