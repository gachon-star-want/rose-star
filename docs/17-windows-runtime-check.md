# Windows 런타임 검증 체크리스트 (원샷)

맥(arm64)에서는 x86 Windows PE를 직접 실행할 수 없어, 게임 로직은 헤드리스 ELF 하네스로
이미 숫자 단언까지 검증했다(`make render-test` / `render-test-loss`). 그러나 **실제 창 표시·
색/방향·마우스/키보드 실동작**은 진짜 Windows에서만 확인된다. 이 문서는 그 한 번의 검증을
빠짐없이 끝내기 위한 체크리스트다.

정적 감사(2026-06-02) 결과 코드레벨 리스크는 낮음: GetDC↔ReleaseDC·BeginPaint↔EndPaint 균형,
모든 import 해결(빌드 에러 0), WM_QUIT 처리, seam 무결(game.inc는 OS API 0회). 남은 건 환경적
검증뿐이다.

## 0. 환경 준비

- **Apple Silicon Mac**: Parallels(ARM Windows 11 + x86 에뮬레이션) 또는 UTM(x86 Windows). wine은
  권위가 없어(에뮬 다층) 사용하지 않는다.
- **Intel Mac / 실 Windows PC**: 그냥 실행하면 된다.
- 빌드: 맥에서 `make build` → `dist/rose-star.exe` 를 VM/PC로 복사.
- **인터넷 차단 상태에서 실행**(제출 환경이 오프라인일 수 있음 — 의존성 누수 검증).

> **갱신(2026-06-15): 게임 완성 상태 반영.** 스프라이트·실제 한글 폰트·모드선택·서사
> 장면·44해넘이·보너스 모드까지 모두 구현·헤드리스 검증됨. 아래는 그것을 실제 Windows에서
> 한 번 확인하는 체크리스트다. `dist/rose-star.exe`는 정확히 1,474,560 B(3.5" 플로피 exact-fill).

## 1. 타이틀 & 표시

- [ ] 창이 뜬다. 크기 **1024×768**(클라이언트), 타이틀바 `The Rose Star`, 리사이즈 불가.
- [ ] **타이틀 화면**: `어린왕자: 장미의 별` 한글 제목 + `새로 시작` + 하단 `쉬움`/`어려움`
      (← → 로 토글, 선택된 쪽 밝게). 글자가 **읽히는 한글**인지(깨진 블록이면 폰트 오류).
- [ ] Space/Enter → **도입 서사**(읽히는 한글 여러 줄, 중앙 정렬) → 한 키 더 누르면 1-1 시작.

## 2. 인게임 표시 (해넘이 1-1)

- [ ] 어두운 청록 지면 + 가로 **모래색 경로(행 6)** + 우측 끝 **빨강 장미 스프라이트** + 빌드존(파랑).
- [ ] 상단 HUD: 좌측 `별빛 N`, 중앙 **빨강 꽃잎 pip 5개**, 우측 **타워 팔레트 5칩**(양 강조).
- [ ] **색 채널**: 장미·꽃잎이 **빨강**(파랑이면 B↔R 오류). **방향**: 경로 가로, 장미 우측, HUD 상단.
- [ ] **스프라이트 확인**: 적이 **연두색 바오밥 싹**, 타워가 **흰 양**으로 보인다(단색 사각형 아님).

## 3. 입력 (D066) — 마우스 + 키보드

- [ ] 마우스 이동 → **커서 테두리**(밝은 노랑)가 칸을 따라온다.
- [ ] 빌드존 **좌클릭** → 선택 타워 설치(별빛 차감), 타워 위 **우클릭** → 판매(+60%).
- [ ] **Tab** 또는 숫자 **1~5** → 타워 종류 전환(해금된 것만). HUD 우측 **팔레트 칩 클릭**으로도 선택.
- [ ] 방향키로 커서 이동(HUD 행으로 안 올라감), **Space**=설치, **S**=판매.
- [ ] **별빛 부족/빌드존 아닌 곳** 클릭 → 설치 안 됨.

## 4. 한 판 ~ 캠페인 흐름

- [ ] 1-1: 왼쪽에서 **싹**이 경로 따라 장미로(8마리). 양 사거리(3칸)에서 처치(+별빛), 누수 시 꽃잎-1.
- [ ] **승리**(8마리 다 막음) → 초록 테두리 + `별을 지켰` → 키 누르면 다음 해넘이(1-2).
- [ ] **달 경계**(1-11 클리어 등) → **막간 서사** 한 장면 → 다음 달 첫 해넘이.
- [ ] **쉬움 모드**면 플레이 중 별빛이 천천히 자동 증가(초당 +2). 어려움은 증가 없음.
- [ ] 후반 해넘이에서 화산·가로등·바람·여우 타워, 새싹/묘목/거목/뱀/허영쟁이/사업가/술꾼 적 등장.

## 5. 재시작 · 일시정지 · 종료

- [ ] 승/패 후 **R** → 같은 해넘이 재시작. **P** → 일시정지(`멈췄다`), 다시 P 해제.
- [ ] **F** → 2배속 토글, **M** → 음소거, **N** → 다음 무리 조기 소환.
- [ ] **ESC** → 창 닫힘, 프로세스 종료(잔류 없음).

## 6. 사운드 · 공유 카드 · 기록 코드

- [ ] 설치/판매/처치/누수/승리/패배에 **사각파 효과음**(winmm). M으로 음소거 토글.
- [ ] 승/패 화면에서 **S** → `share/<코드>.bmp` 480×480 공유 카드 저장(`카드를 남겼다`).
- [ ] 타이틀에서 **I** → 기록 코드 입력(`RS-달-해넘이-모드-HEX`) → 해당 지점부터 이어하기.

## 7. 보너스 모드 (해금 후)

- [ ] 2-11 클리어 후 타이틀에 `점등인의 별 [E]` 표시 → **E** → 무한 웨이브(하단 `무리 N` 점수 증가).
- [ ] 3-11 클리어 후 `사막의 신기루 [D]` → **D** → 매번 다른 경로의 맵 1판.

## 8. 보고 형식 (실패 시)

각 실패 항목을 `[섹션.항목] 기대 → 실제` 로 적어주면 바로 수정한다.
예: `[2] 장미가 파랑 → B↔R 채널`, `[1] 제목이 깨진 블록 → 폰트`, `[3] Tab으로 타워 안 바뀜`.

## 검증 범위 밖 (알려진 한계)

- 프레임 페이싱은 `Sleep(16)` 근사(QPC 고정스텝 아님) — 체감 60fps 근처면 정상.
- 보너스 모드 밸런스는 헤드리스로 "동작/생성 유효"만 확인(캠페인처럼 lock된 수치 아님).
