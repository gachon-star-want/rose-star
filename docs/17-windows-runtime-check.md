# Windows 런타임 검증 체크리스트 (원샷)

맥(arm64)에서는 x86 Windows PE를 직접 실행할 수 없어, 게임 로직은 헤드리스 ELF 하네스로
이미 숫자 단언까지 검증했다(`make render-test` / `render-test-loss`). 그러나 **실제 창 표시·
색/방향·마우스/키보드 실동작**은 진짜 Windows에서만 확인된다. 이 문서는 그 한 번의 검증을
빠짐없이 끝내기 위한 체크리스트다.

정적 감사(2026-06-02) 결과 코드레벨 리스크는 낮음: GetDC↔ReleaseDC·BeginPaint↔EndPaint 균형,
모든 import 해결(빌드 에러 0), WM_QUIT 처리, seam 무결(game.inc는 OS API 0회). 남은 건 환경적
검증뿐이다.

## 0. 환경 준비

- **Apple Silicon Mac**: Parallels(ARM Windows 11 + x86 에뮬레이션) 또는 UTM(x86 Windows). wine은
  권위가 없어(에뮬 다층) 사용하지 않는다.
- **Intel Mac / 실 Windows PC**: 그냥 실행하면 된다.
- 빌드: 맥에서 `make build` → `dist/rose-star.exe` 를 VM/PC로 복사.
- **인터넷 차단 상태에서 실행**(제출 환경이 오프라인일 수 있음 — 의존성 누수 검증).

## 1. 창 & 표시

- [ ] 창이 뜬다. 크기 **1024×768**(클라이언트), 타이틀바 텍스트 `The Rose Star`.
- [ ] 리사이즈 불가(테두리 드래그로 안 늘어남 — WS_CAPTION/SYSMENU/MINIMIZE만).
- [ ] 화면이 **검게/깨져 보이지 않고** 맵이 그려진다:
  - 어두운 청록 지면 + 가로 **모래색 경로(행 6)** + 우측 끝 **빨강 장미** + 빌드존 4칸(파랑 테두리).
  - 상단 HUD 바: 좌측 **노랑 별빛 막대**, 우측 **빨강 꽃잎 pip 5개**.
- [ ] **색 채널 검증**: 장미·꽃잎이 **빨강**으로 보이는가(파랑으로 뒤바뀌면 B↔R 채널 오류).
- [ ] **방향 검증**: 경로가 가로, 장미가 **우측**, HUD가 **상단**(상하 뒤집힘이면 top-down DIB 오류).

## 2. 마우스 입력 (D066)

- [ ] 마우스를 움직이면 **커서 하이라이트 테두리**(밝은 노랑)가 칸을 따라온다.
- [ ] 빌드존 칸에 **좌클릭** → 흰 양이 설치되고 별빛 막대가 줄어든다(20).
- [ ] 양 위에 **우클릭** → 양이 사라지고 별빛이 일부 환급된다(+12).
- [ ] 별빛이 20 미만일 때 좌클릭 → 설치 안 됨(돈 부족).
- [ ] 경로/지면(빌드존 아님)에 좌클릭 → 설치 안 됨.

## 3. 키보드 입력 (D066)

- [ ] 방향키 ←↑→↓ 로 커서가 한 칸씩 이동(상단 HUD 행으로는 안 올라감).
- [ ] **Space** = 설치, **S** = 판매 (마우스와 동일 동작).

## 4. 한 판 플레이 (해넘이 1-1)

- [ ] 가만히 두면 **왼쪽에서 연두색 싹**이 등장해 경로를 따라 **오른쪽 장미로** 이동한다(총 8마리).
- [ ] 양 사거리(3칸) 안에 싹이 들어오면 잠시 후 싹이 **사라지고**(처치) 별빛이 조금씩 오른다(+1/처치).
- [ ] 양이 없거나 못 막으면 싹이 장미에 도달 → **꽃잎 pip이 하나 줄어든다**.
- [ ] **승리**: 양을 적절히 깔아 8마리를 다 막으면 → 화면 가장자리에 **초록 테두리 프레임**.
- [ ] **패배**: 일부러 비워 꽃잎 5개를 다 잃으면 → **빨강 테두리 프레임**, 게임 정지.

## 5. 재시작 & 종료

- [ ] 승/패 후 **R** 키 → 새 판이 처음부터 시작(별빛 60, 꽃잎 5, 적 리셋).
- [ ] **ESC** → 창이 닫히고 프로세스 종료(잔류 프로세스 없음).

## 6. 보고 형식 (실패 시)

각 실패 항목을 `[섹션.항목] 기대 → 실제` 로 적어 돌려주면 바로 수정한다. 예:
`[1] 장미가 파랑으로 보임 → B↔R 채널 의심` / `[4] 싹이 오른쪽이 아니라 위로 감 → 경로/좌표 의심`.

## 알려진 미구현(이 검증 범위 아님)

- Easy/Hard 모드 별빛 차이(Easy +2/초) — Track C 잔여.
- 결과 텍스트("별을 지켰다/별이 어두워졌다") — 서브셋 폰트 도입 후. 지금은 색 프레임으로 대체.
- 실제 스프라이트 아트 — 지금은 색 도형. cast24 PNG→asm 변환은 다음 단계(D078).
- 프레임 페이싱은 `Sleep(16)` 근사 — QPC 고정스텝은 후순위.
